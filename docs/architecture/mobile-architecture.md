# Mobile Architecture

## 1. Purpose

Мобильное приложение для стройконтроля в полевых условиях.

Ключевое требование: **полная функциональность офлайн**.

---

## 2. Stack

React Native + TypeScript + WatermelonDB (ADR-008) + SQLite + Expo Camera / геолокация + FCM/APNs. Приложение: `apps/mobile/`.

---

## 3. Offline-First Architecture

### WatermelonDB (ADR-008)

Локальная база данных: зеркало серверных данных + очередь операций.

- Все записи создаются локально → синхронизируются при появлении сети
- Очередь операций персистентна (survives app restart)
- Индикация статуса синхронизации пользователю

### Операции в офлайн-режиме

| Операция | Офлайн | Замечание |
|----------|--------|-----------|
| Создание верификации | ✓ | полностью офлайн |
| Фотофиксация | ✓ | хранится локально, загружается при sync |
| Ввод данных по позициям | ✓ | очередь |
| Просмотр периода | ✓ | из локального кэша |
| Закрытие периода | ✗ | требует подтверждения сервера |

---

## 4. Sync Protocol

### Синхронизация

- Фоновая синхронизация при появлении сети
- Retry с экспоненциальным backoff при ошибках
- Каждая операция идемпотентна (повторная отправка безопасна)

### Конфликт-резолюция (ADR-003)

- Timestamp-based: побеждает более поздняя запись
- Server-wins для критических полей (статус периода, финансовые данные)
- Неразрешимые конфликты → показать UI для ручного выбора

---

## 5. Фотофиксация

- Expo Camera + Expo Location
- EXIF метаданные: timestamp + GPS координаты + точность GPS
- Сжатие до < 2 MB перед загрузкой, оригинал хранится локально
- Загрузка на S3 при синхронизации с прогрессом

---

## 6. Push Notifications (ADR-014)

- FCM для Android, APNs для iOS
- Типы уведомлений: SLA алерт, смена статуса периода, новое расхождение
- Уведомления отправляются через BullMQ очередь на сервере (не напрямую)
- Обработка в foreground + background + terminated state
