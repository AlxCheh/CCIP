# Claude.md — инструкция проектного агента CCIP

## 1. Контекст проекта

**CCIP** (Construction Control & Intelligence Platform) — SaaS-платформа для оцифровки и аналитики объектов капитального строительства.

**Ключевая ценность:** Заказчик видит верифицированный факт выполнения работ, а не данные ГП. Единственный оператор системы — строительный контроль Заказчика.

Главный архитектурный документ: `docs/architecture_v1_0.md`

## 6. Правило чтения архитектуры при оптимизации документов

**При запросе на оптимизацию / редактирование конкретного документа — читать только нужный раздел, а не файл целиком.**

Карта разделов `architecture_v1_0.md`:

|Запрос касается…|Читать раздел|
|---|---|
|Общая архитектура, стиль, модули|§1|
|Уровни L0–L3, межмодульные зависимости|§2|
|Backend, сервисные слои, бизнес-правила, SLA Scheduler, GP-токен|§3|
|Web / Mobile / GP Form, экраны, offline-first, state management|§4|
|Схема БД, инварианты, потоки данных, кросс-версионность|§5|
|Внешние интеграции, форматы данных, BIM|§6|
|Окружения, деплой, партиционирование, MV, sync_queue lifecycle|§7|
|Auth, RBAC-матрица, защита данных, целостность|§8|
|Sequence-диаграммы (период, SLA, baseline, sync)|§9|
|Риски R-01..R-11, узкие места, пробелы §10.4|§10|
|Реестр ADR, соответствие патчам схемы|§11|
|Сводка компонентов и статусов|конец файла|

**Workflow для задачи "оптимизировать раздел X":**

1. Прочитать соответствующий раздел `architecture_v1_0.md` (через `view` с `view_range`).
2. Прочитать упомянутые в разделе ADR (только их).
3. Проверить `errors_log.md` на известные проблемы по теме.
4. Внести изменения. Не трогать соседние разделы без явного запроса.
